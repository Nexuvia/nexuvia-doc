# React — Assembly Guide

This guide covers wiring Nexuvia in a **client-side React SPA** (Vite or Create React App). There is no SSR — all rendering happens in the browser.

:::warning Read this first
**A pure React SPA cannot run Nexuvia by itself.** Two things require a server:

1. **`@nexuvia/auth-server`** — must hold the OAuth `client_secret` (browser cannot)
2. **`@nexuvia/cart`** — SAP OCC blocks browser calls with CORS

You need a thin backend (Express, Hono, Fastify, or any other Node.js framework) to host the route handlers. The React app calls your backend, which calls SAP. **Nexuvia in pure SPA mode = mock adapters only.**

If you want full SAP integration with no separate backend, use [Next.js](/frameworks/nextjs) instead.
:::

## What works in pure SPA mode

| Library | Pure SPA | Needs backend |
|---------|---------|---------------|
| `@nexuvia/core` | ✅ | — |
| `@nexuvia/log` | ✅ | — |
| `@nexuvia/storage` | ✅ | — |
| `@nexuvia/cms` (with `MockCmsAdapter`) | ✅ | OCC adapter needs server |
| `@nexuvia/product` (with `MockProductAdapter`) | ✅ | OCC adapter needs server |
| `@nexuvia/search` (with `MockSearchAdapter`) | ✅ | OCC adapter needs server |
| `@nexuvia/analytics` (GTM) | ✅ | — |
| `@nexuvia/smartedit` | ✅ | — |
| `@nexuvia/cart` | ❌ | Always — CORS |
| `@nexuvia/auth-server` | ❌ | Always — secret can't leak |
| `@nexuvia/auth-client` | ❌ | Always — needs callback route |

## Step 1 — Vite env vars

Vite replaces env vars at build time using `import.meta.env`. Prefix all client-visible vars with `VITE_`:

```bash
# .env
VITE_HYBRIS_HOST=api.commerce.example.com
VITE_BASE_SITE=my-basesite
VITE_GTM_ID=GTM-XXXXXXX
VITE_API_BASE=http://localhost:3001          # your backend URL
```

```ts
// nexuvia.config.ts
import type { NexuviaConfig } from '@nexuvia/nexuvia';

const config: NexuviaConfig = {
  hybris: {
    protocol:    'https',
    host:        import.meta.env.VITE_HYBRIS_HOST || '',
    port:        '',
    version:     'v2',
    occBasePath: '/occ',
    cmsBasePath: '/customws',
  },
  stores: {
    default: {
      baseSite:           import.meta.env.VITE_BASE_SITE || 'my-site',
      domain:             'localhost',
      supportedLanguages: ['en'],
      defaultLanguage:    'en',
      currency:           'USD',
      country:            'US',
      isEcommerce:        true,
    },
  },
  cms:        { useMock: true, pageLabels: { homepage: 'homepage', productDetail: 'productDetails',
                cart: 'cartPage', search: 'search', searchEmpty: 'searchEmpty', notFound: 'notFound' },
                componentTypes: [] },
  smartedit:  { allowedOrigins: [], previewVersion: 'v1' },
  authServer: { clientId: '', clientSecret: '', tokenEndpoint: '' },
  authClient: { session: { encryptionKey: '', secureCookies: false },
                storeConfigProvider: async (storeKey) => getAzureStoreConfig(storeKey) },
  analytics:  { gtmContainerId: import.meta.env.VITE_GTM_ID || '' },
};

export default config;
```

:::danger Never put secrets in VITE_*
Anything starting with `VITE_` is **bundled into your JavaScript and visible to every user**. Never put `OAUTH_CLIENT_SECRET` or `AUTH_ENCRYPTION_KEY` here. They belong in the backend's `.env`.
:::

## Step 2 — Single-file client instantiation

Construct all clients **once** in a single module so the same instance is reused across the app:

```ts
// src/lib/clients.ts
import { OccClient }                            from '@nexuvia/occ';
import { CookieStorage }                        from '@nexuvia/storage';
import { CartClient, ProxyCartAdapter }         from '@nexuvia/cart/client';
import { ProductClient, OccProductAdapter, MockProductAdapter } from '@nexuvia/product';
import { SearchClient, OccSearchAdapter, MockSearchAdapter }    from '@nexuvia/search';
import { CmsClient, MockCmsAdapter }            from '@nexuvia/cms/server';
import { GtmAnalyticsAdapter, AnalyticsClient } from '@nexuvia/analytics';
import config from '../../nexuvia.config';

const { hybris, stores } = config;
const store      = stores.default;
const baseUrl    = `${hybris.protocol}://${hybris.host}`;
const occConfig  = { baseUrl, basePath: hybris.occBasePath, version: hybris.version };

const occClient = new OccClient(occConfig, store.baseSite, store.defaultLanguage);
const apiBase   = import.meta.env.VITE_API_BASE || '';

// CART — always proxy through your backend
export const cartClient = new CartClient(
  new ProxyCartAdapter({
    baseSite: store.baseSite,
    language: store.defaultLanguage,
    apiBase:  `${apiBase}/api/cart`,
  }),
  new CookieStorage(),
);

// PRODUCT — mock in dev, OCC via your backend in prod
// Note: OccProductAdapter calls SAP directly — only works if your backend proxies to SAP
// Otherwise wrap a custom FetchProductAdapter that calls your backend
export const productClient = new ProductClient(
  import.meta.env.DEV
    ? new MockProductAdapter()
    : new OccProductAdapter(occClient),
);

// SEARCH — same pattern as product
export const searchClient = new SearchClient(
  import.meta.env.DEV
    ? new MockSearchAdapter()
    : new OccSearchAdapter(occClient),
);

// CMS — always mock in pure SPA (CMS endpoint requires OAuth)
export const cmsClient = new CmsClient(
  new MockCmsAdapter(async (label) => {
    const res = await fetch(`/mock/${label}.json`);
    return res.ok ? res.json() : null;
  }),
);

// ANALYTICS
export const analyticsClient = new AnalyticsClient(
  new GtmAnalyticsAdapter({ containerId: config.analytics.gtmContainerId }),
);
```

## Step 3 — Provider tree in `App.tsx`

Same nesting order as Next.js (see [Next.js guide](/frameworks/nextjs)):

```tsx
// src/App.tsx
import { Suspense } from 'react';
import { GtmScript } from '@nexuvia/analytics';
import { StoreProvider }     from '@/store/store-context';
import { AuthProvider }      from '@/providers/auth-provider';
import { CartProvider }      from '@/providers/cart-provider';
import { AnalyticsProvider } from '@/providers/analytics-provider';
import { Router }            from '@/router';

import { cartClient, analyticsClient } from '@/lib/clients';
import config from '../nexuvia.config';

const store = config.stores.default;

export function App() {
  return (
    <>
      {config.analytics.gtmContainerId && (
        <GtmScript containerId={config.analytics.gtmContainerId} />
      )}
      <StoreProvider storeKey="default" storeConfig={store} language={store.defaultLanguage}>
        <AuthProvider>
          <CartProvider client={cartClient}>
            <AnalyticsProvider client={analyticsClient}>
              <Suspense fallback={null}>
                <Router />
              </Suspense>
            </AnalyticsProvider>
          </CartProvider>
        </AuthProvider>
      </StoreProvider>
    </>
  );
}
```

## Step 4 — Use in pages

```tsx
// src/pages/ProductPage.tsx
import { useEffect, useState } from 'react';
import { useCart } from '@/providers/cart-provider';
import { productClient } from '@/lib/clients';

export function ProductPage({ code }: { code: string }) {
  const [product, setProduct] = useState(null);
  const { addItem } = useCart();

  useEffect(() => {
    productClient.getProduct(code).then(setProduct);
  }, [code]);

  if (!product) return <p>Loading…</p>;
  return (
    <div>
      <h1>{product.name}</h1>
      <button onClick={() => addItem(product.code)}>Add to Cart</button>
    </div>
  );
}
```

## Backend setup — minimal Express example

```ts
// server/index.ts
import express from 'express';
import cors    from 'cors';
import { createOccClient } from './config/server';
import { OccCartAdapter }  from '@nexuvia/cart/server';

const app = express();
app.use(cors({ origin: 'http://localhost:5173', credentials: true }));
app.use(express.json());

app.post('/api/cart', async (req, res) => {
  const { cartId, baseSite, language, productCode, quantity = 1 } = req.body;
  const client  = createOccClient(baseSite, language);
  const adapter = new OccCartAdapter(client);
  // addToCart takes two args: cartId and payload with productCode
  // res.json(await adapter.addToCart(cartId ?? null, { productCode, quantity }));
});

app.listen(3001, () => console.log('Backend on :3001'));
```

The route handler logic is identical to the [Next.js cart route](/wiring/cart) — just expressed in your backend framework's syntax.

## Common errors

### `process is not defined` in browser

You imported `nexuvia.config.ts` and it references `process.env.X`. Use `import.meta.env.VITE_X` instead.

### CORS error on `/api/cart`

Backend isn't sending `Access-Control-Allow-Origin`. Add `cors()` middleware to your backend.

### `useCart() must be used inside <CartProvider>`

Provider tree missing — check `App.tsx`.

### Mock CMS returns 404

Mock JSON not in `public/mock/`. Vite serves files from `public/` at the root URL — `MockCmsAdapter` should fetch `/mock/homepage.json`.

## Checklist

- [ ] `nexuvia.config.ts` uses `import.meta.env.VITE_*`, never `process.env`
- [ ] No `OAUTH_CLIENT_SECRET` or encryption key in any `VITE_*` var
- [ ] Backend running for cart + auth (or stick to mock-only for demos)
- [ ] All clients instantiated **once** in `src/lib/clients.ts`
- [ ] Provider tree in `App.tsx` matches Next.js order: Store → Auth → Cart → Analytics → Suspense
- [ ] CORS configured on backend for the React dev server origin
